# 需求分析：优坐标招募合伙人线索闭环

## 1. 背景与范围

本稿单独分析优坐标“招募与合伙人线索闭环”，覆盖小程序招募入口、后台线索、跟进、关闭或转为渠道商。输入来源为 Axure 原型中的 `申请合伙人列表（2026_6_5）`、`招募老师列表`、`渠道商管理`、`我的`、`创客中心`、历史 `原优化界面`，以及 v1/v2 需求分析稿。

目标是把“线索”与“正式渠道商/教师/创客”拆清楚：线索阶段负责收集和跟进，转化后进入渠道商名额闭环或教师入驻闭环。本稿不输出 API、数据库或开发实现。

关键结论：

- 已确认需求：原型中存在招募老师列表和历史申请合伙人列表；申请合伙人列表支持添加跟进、关闭、转为渠道商。
- 推测需求：小程序招募合伙人、招商加盟、邀请老师入驻可能进入不同后台列表，但入口到列表映射未完全证明。
- 待确认需求：申请合伙人列表是否继续沿用 `2026_6_5` 版本、转为渠道商后是否自动启用账号和初始名额。
- 暂不纳入范围：正式渠道商名额运营和会员开通，归属渠道商名额闭环。

## 2. 业务规则

- 小程序可能存在招募合伙人、招商加盟、邀请老师入驻等入口，提交后进入对应后台线索列表。
- `招募老师列表` 用于老师招募线索跟进，是否进入教师申请审核需确认。
- `申请合伙人列表（2026_6_5）` 是历史版本页面，但原型中可见线索跟进、关闭、转为渠道商能力，可作为合伙人线索闭环参考。
- 合伙人线索可添加跟进记录，记录跟进方式、跟进内容、最新跟进时间和下次跟进时间。
- 合伙人线索可关闭，关闭后是否可重新开启待确认。
- 合伙人线索可转为渠道商；原型说明转为渠道商时联系人、电话带入，渠道商名称默认为联系人名称。
- 转为渠道商后进入渠道商名额闭环，后续名额充值、开通会员和转让不在本稿展开。

## 6. PC/移动端差异

PC 端：

- 后台负责线索查看、跟进、关闭和转化。
- 申请合伙人列表当前来自历史日期版本，是否作为当前后台页面需确认。

移动端：

- 小程序负责招募老师、招募合伙人、招商加盟等入口提交。
- 具体入口页面和表单字段未完全在当前有效页面确认。

历史/暂废：

- `申请合伙人列表（2026_6_5）` 只作为历史版本参考，不默认进入当前有效主链路。
- `原优化界面` 中招商加盟入口只作为来源说明。

## 8. 页面与跳转逻辑

```text
小程序招募/招商入口提交线索
-> 后台申请合伙人列表或招募老师列表形成线索
-> 运营添加跟进记录
-> 线索无效则关闭
-> 合格合伙人线索转为渠道商
-> 渠道商管理生成渠道商记录
-> 进入渠道商名额闭环
```

老师线索支线：

```text
邀请老师入驻/二维码提交
-> 招募老师列表跟进
-> 是否进入教师申请审核待确认
-> 进入教师入驻与师资管理闭环
```

## 9. 待确认问题

- 申请合伙人列表是否继续沿用 `2026_6_5` 版本。
- 招募合伙人、招商加盟、邀请老师入驻分别进入哪个后台列表。
- 转为渠道商后是否自动启用渠道商账号，是否需要初始名额。
- 线索关闭是否需要原因，关闭后是否可重新开启。
- 创客中心“立即申请”提交后是否进入申请合伙人列表，还是进入其他审核流程。

## 10. 风险与依赖

- 如果历史申请列表不再使用，当前合伙人线索闭环缺少有效后台承接页面。
- 转渠道商依赖渠道商名额闭环，若初始账号和名额规则不清，会造成转化后无法继续运营。
- 招募老师线索与教师申请审核关系不清，会影响师资入驻流程。

## 11. 下一步动作

- 业务流程阶段分别画合伙人线索跟进、关闭、转渠道商，以及老师线索跟进流程。
- 信息架构阶段确认申请合伙人列表是否进入当前 PC 后台导航。
- 交互设计阶段补齐跟进弹窗、关闭确认、转渠道商弹窗、重复手机号提示。

## 12. 原型设计输入

- 页面范围：申请合伙人列表历史参考、招募老师列表、创客中心申请入口、渠道商管理转化结果。
- 字段批注：用户信息、电话、地区、类型、跟进记录、状态、联系人、渠道商名称。
- 按钮批注：添加跟进、关闭、转为渠道商、立即申请。
- 黄色说明区：历史页面是否沿用、入口映射、转渠道商后账号和初始名额待确认。
