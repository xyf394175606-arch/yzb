# 需求分析：优坐标创客分佣闭环

## 1. 背景与范围

本稿单独分析优坐标“创客分佣闭环”，覆盖注册拉新、课程分享成交、教师课消师资分佣、佣金生成、团队/明细展示和待结算/可提现状态。输入来源为 Axure 原型中的 `创客中心`、`分佣比例设置`、`分佣账单`、`家长管理`、`账号管理`，以及 v1/v2 需求分析稿。

目标是把创客中心中的收益、团队、分佣明细与提现入口拆成分佣闭环本身，提现处理另见 `需求分析_优坐标提现结算闭环_v1_20260629.md`。本稿不输出 API、数据库或开发实现。

关键结论：

- 已确认需求：创客中心展示累计收益、可提现、待结算、团队、佣金明细和团队动态；后台有分佣比例设置和分佣账单。
- 推测需求：三套分佣链路都进入当前版本，但仍需业务和财务确认。
- 待确认需求：分佣比例/固定金额最终口径、二星团员对家长/教师的展示方式、团队动态是否展示访问记录。
- 暂不纳入范围：真实结算、打款凭证和提现状态处理，归属提现结算闭环。

## 2. 业务规则

- 创客中心原型明确分销分为三套独立链路：注册拉新、课程分享成交、教师课消师资，分开计算、分开生成佣金流水。
- 注册拉新二级间推仅限合伙人领取；课程分享二级、师资课消二级不限制身份，家长、教师、合伙人均可领取。
- 注册上级与课程分享人为同一人时，两套成交链路佣金可叠加发放；不同人则各自领取自身链路一级佣金，并按各自链路生成二级佣金。
- 师资分佣链路独立于注册推荐人和课程分享人；多教师排课按课时占比拆分师资一、二级佣金。
- 后台 `分佣比例设置` 可配置注册直接推荐人、注册间接推荐人、购课直接推荐人、课消教师、课消教师推荐人等分佣关系。
- 分佣计算模式包括固定金额和按比例；分佣比例或固定分佣金额可编辑，分佣规则可启用/禁用。
- `分佣账单` 记录流水号、源订单号、付费用户、佣金受益人、收益类型、订单支付金额、收益金额、状态和生成时间。
- 可提现金额进入提现结算闭环，本稿只说明分佣如何生成和展示。

## 6. PC/移动端差异

PC 端：

- 后台负责分佣规则配置、分佣流水查看、推荐关系和账号状态查询。
- 后台不承担用户收益展示体验，但其规则决定小程序收益结果。

移动端：

- 小程序 `创客中心` 展示收益、团队、明细和动态，是创客查看结果和发起提现的入口。
- 家长、教师、合伙人可能看到不同团队层级和二星团员展示方式。

历史/暂废：

- 暂无历史分佣页面进入主链路；旧版入口只作为来源说明。

## 8. 页面与跳转逻辑

```text
后台分佣比例设置配置注册/购课/课消规则
-> 用户注册拉新、课程分享成交或教师课消触发分佣
-> 后台分佣账单生成流水
-> 创客中心汇总累计收益、待结算、可提现和明细
-> 用户查看团队、成员详情和佣金贡献
-> 可提现金额达到条件后进入提现结算闭环
```

## 9. 待确认问题

- 注册拉新、课程分享成交、教师课消三套分佣链路是否全部进入当前版本。
- 分佣比例和固定金额以后台配置为准，还是需要财务确认后固化。
- 家长和教师是否隐藏二星团员，还是置灰展示并提示不可产生收益。
- 团队动态是否展示访问记录；原型建议后续通过消息推送关键行为。
- 账号关闭后团队关系、历史佣金和新增佣金如何处理。

## 10. 风险与依赖

- 分佣规则涉及财务结算，未确认前不能作为最终 PRD 结论。
- 注册、课程订单和排课课消分别来自客户、课程售卖和排课闭环，任一上游状态不清都会影响佣金生成。
- 创客中心同时承载提现入口，若不拆分会混淆分佣生成和提现处理。

## 11. 下一步动作

- 业务流程阶段分别画注册拉新分佣、课程分享分佣、教师课消分佣三条流程。
- 信息架构阶段将创客中心拆为收益总览、团队、明细、提现入口四类信息。
- 交互设计阶段补齐无收益、无团队、规则禁用、账号关闭、二星团员展示差异。
