# 需求分析：个人中心入口权限控制

## 1. 背景与范围

本次需求基于 `需求分析_个人中心入口权限控制_v1_20260622.md` 的待确认问题进行补充确认，主要解决后台需要按单个家长或教师用户控制其个人中心中不同业务入口是否展示的问题。移动端涉及家长端、教师端个人中心；PC 后台涉及用户管理中的家长管理、教师管理用户详情。本次只整理入口展示权限的需求分析，不直接修改 `需求分析_分销模块整体需求_v4_20260622.md`，不输出原型，不写接口字段、数据库字段或技术实现。

输入来源：

| 来源 | 用途 |
| --- | --- |
| 用户补充需求 | 明确后台可控制家长端或教师端单个用户个人中心的分享入口、分销入口展示 |
| 图 1：家长端个人中心截图 | 识别个人中心中已可见的入口、创客中心卡片、服务区和底部导航 |
| 图 2：后台家长管理截图 | 识别当前后台用户管理页面结构、用户详情和操作入口 |
| `05_需求分析/需求分析_分销模块整体需求_v4_20260622.md` | 作为创客中心/分销入口权限与分销权限区分的上游依据 |
| 用户本轮确认事项 20260623 | 明确 v1 中 10 项待确认问题的处理结论 |
| 新版 `requirement-analysis` skill | 页面清单列顺序、内容展示属性、按钮与弹窗规则 |

关键结论：

- 本需求新开专项需求分析文件，作为同版本新增功能，不直接覆盖或并入分销整体需求 v4。
- 后台控制粒度按多个入口开关整理，包括申请加盟、邀请家长报名、邀请老师入驻、招募合伙人、创客中心/分销入口。
- 入口权限控制按单个用户生效，覆盖家长用户和教师用户。
- 入口展示权限与分销权限审核是两个概念：入口是否展示不等于用户是否已开通分销能力。
- 家长端和教师端个人中心使用完全相同的入口名称。
- 全部分享类入口关闭时，移动端个人中心入口区整体隐藏。
- 创客中心/分销入口关闭时，不允许用户通过其他路径进入创客中心。
- 后台入口权限配置采用弹窗；保存时必须填写变更原因，并展示配置人、配置时间、操作日志或历史版本。
- 新用户默认入口展示规则来自角色配置；入口权限变更后，用户刷新个人中心即生效。
- 截图中的可见内容只作为现有页面事实；后台入口权限开关是本次新增需求，不是截图已证明的现有能力。

## 5. PC/移动端差异

移动端：

- 家长端和教师端个人中心展示受后台控制的入口。
- 家长端和教师端使用完全相同的入口名称。
- 移动端只展示入口，不提供入口权限编辑能力。
- 被关闭的入口不展示；被打开的入口展示并沿用原入口点击路径。
- 全部分享类入口关闭时，个人中心入口区整体隐藏。
- 创客中心/分销入口关闭时，不允许用户通过其他路径进入创客中心；入口展示后，仍按分销权限状态决定进入、申请、等待审核或拒绝展示。
- 入口权限变更后，用户刷新个人中心即生效。

PC 后台：

- 家长管理和教师管理需要支持对单个用户配置入口展示权限。
- 后台配置操作对象是当前选中的用户，不是列表批量操作。
- 后台入口权限配置采用弹窗。
- 保存入口权限必须填写变更原因，并展示配置人、配置时间、操作日志或历史版本。
- 新用户默认入口展示规则来自角色配置。

## 9. 原型设计输入

### 原型范围

移动端：

- 涉及：家长端个人中心、教师端个人中心。
- 需要画的页面：个人中心。
- 需要覆盖的状态：入口展示、入口隐藏、创客中心入口展示但分销权限未开通/待审核/被拒绝/已开通。
- 需要覆盖的规则：全部分享类入口关闭时入口区整体隐藏；创客中心/分销入口关闭时不允许通过其他路径进入创客中心；刷新个人中心即按最新配置生效。

PC 端：

- 涉及：家长管理-用户详情、教师管理-用户详情、入口权限设置弹窗。
- 需要覆盖的状态：有权限编辑、无权限隐藏按钮、保存成功、保存失败、取消后不变、变更原因未填不可保存。
