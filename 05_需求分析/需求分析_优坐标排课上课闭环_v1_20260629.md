# 需求分析：优坐标排课上课闭环

## 1. 背景与范围

本稿单独分析优坐标“排课上课闭环”，覆盖购课/试听/班课对象形成、后台排课、课程表展示、签到、扣课时和上课记录回流。输入来源为 Axure 原型中的 `一对一`、`试听课`、`班课管理`、`签到`、`我的课程表`、`教师详情`、`家长查看详情页`，以及 v1/v2 需求分析稿。

目标是把教务后台操作和小程序课程表展示拆成一个完整履约闭环，供后续业务流程、信息架构和交互设计直接使用。本稿不输出接口、数据库或技术实现。

关键结论：

- 已确认需求：一对一、试听课、班课管理均支持排课；签到提交后不可修改，已签到扣课时，请假不扣；距开课 5 分钟内或开课后不可编辑/删除课节。
- 推测需求：排课与 ClassIn 存在强关联，但接口失败降级未明确。
- 待确认需求：ClassIn 是否强依赖、补签/撤回规则、移出班级后的课时处理。
- 暂不纳入范围：占位 `课程表` 页面未见明确内容，不作为当前闭环主页面。

## 2. 业务规则

- 一对一排课页面以课程购买记录为主，同一学生购买不同课程展示多条；多次购买同一课程合并展示并叠加课时数。
- 一对一课程状态：剩余课时大于 0 为在读，剩余课时为 0 或以下为结课。
- 试听课页面展示购买试听课的学生，已约课时大于 0 为已试听，反之为未试听。
- 排课弹窗中课节名称默认填充课程名称且可编辑，用于 ClassIn 接口；科目默认填充课程科目；授课老师可由购买指定老师课程自动填充并允许更换。
- 年级下拉应只展示课程包含的年级；课时数默认 1，选项为 1、2、3。
- 距离开课时间 5 分钟之前课节可编辑和删除；5 分钟之内或开课时间后不可编辑/删除。
- 班课管理支持添加班级、学生管理、课节管理、排课、删除；学生管理包含添加学生、移出班级；课节管理规则与排课规则一致。
- 签到页面学生状态默认为已签到；如未上课可选择请假；提交后不可修改。已签到学生扣除当前课节课时数，请假学生不扣课时。
- 上课记录回流到教师详情和家长查看详情页，并可能触发评价入口和师资分佣。

## 6. PC/移动端差异

PC 端：

- 后台教务负责排课、班级管理、课节编辑/删除和签到扣课时。
- 后台以列表、弹窗和详情记录为主。

移动端：

- 小程序 `我的课程表` 展示课节、上课入口、签到/评价状态。
- 移动端是否支持教师端操作未明确，目前只按用户查看课程表处理。

历史/暂废：

- `课程表` 后台页面存在但未见明确内容，暂不作为当前闭环有效页面。

## 8. 页面与跳转逻辑

```text
课程订单/试听订单/班级学生形成可排课对象
-> 后台一对一/试听课/班课管理排课
-> 生成课节和已约课时
-> 小程序我的课程表展示课节
-> 用户前往 ClassIn 或查看课程安排
-> 后台签到提交
-> 已签到扣课时，请假不扣课时
-> 上课记录进入教师详情和家长查看详情页
-> 达到条件后进入评价闭环
```

## 9. 待确认问题

- ClassIn 是否为排课强依赖；接口失败时是否允许后台手工保存课节。
- 距离开课 5 分钟内不可编辑/删除规则是否适用于一对一、试听课、班课全部场景。
- 签到提交后是否绝对不可修改，还是管理员可补签/撤回。
- 移出班级学生时剩余课时如何处理。
- 后台 `课程表` 页面是否只是占位，还是存在当前未抽取内容。
- 教师端是否有独立课表、上课或签到能力。

## 10. 风险与依赖

- 排课对象依赖课程售卖闭环和会员/课时规则。
- 授课老师依赖教师入驻与师资管理闭环。
- 签到扣课时会影响会员权益和课程剩余课时。
- 上课完成后可能触发评价闭环和创客分佣闭环。

## 11. 下一步动作

- 业务流程阶段拆出一对一排课、试听课排课、班课排课、签到扣课时四条流程。
- 信息架构阶段确认班级学生管理、课节列表、签到是否作为独立页面或弹窗。
- 交互设计阶段补齐 5 分钟限制、ClassIn 失败、请假、补签、移出班级确认。

- 字段批注：购买课时、已约课时、剩余课时、课节名称、授课老师、课时数、签到状态。
- 按钮批注：排课、已约课时、编辑课节、删除课节、添加班级、添加学生、移出班级、提交签到、上课入口。
- 黄色说明区：ClassIn、5 分钟限制、签到后不可修改、已签到扣课时、请假不扣课时。
