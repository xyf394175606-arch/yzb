# 需求分析：优坐标会员权益闭环

## 1. 背景与范围

本稿单独分析优坐标“会员权益闭环”，覆盖会员开通、续费、升级、权益生效、折扣课时消耗、到期/失效和后台管理。输入来源为优坐标 Axure 原型、v1 页面拆分稿、v2 业务闭环稿和上游 MVP 需求分析。

本闭环目标是把普通会员、卓越会员、渠道商开通会员、折扣课时和后台会员管理拆清楚，为课程售卖、渠道商名额、排课课时和后续原型设计提供边界。本稿不输出接口、数据库或开发实现。

关键结论：

- 已确认需求：小程序 `我的会员权益` 展示会员等级、到期时间、折扣课时和续费/升级入口；后台 `会员管理` 展示会员权益包并支持充值记录、消耗明细和赠送折扣课时。
- 推测需求：小程序开通/续费支付页部分可能来自历史页面，当前主页面未完全证明。
- 待确认需求：会员价格、续费课时叠加规则、卓越会员是否仍需要折扣课时、渠道商开通的会员类型。
- 暂不纳入范围：历史会员页面中未确认回流的支付和展示细节。

## 2. 业务规则

- 小程序会员分为普通会员和卓越会员，原型显示普通会员 `499元/年`、卓越会员 `1000元/年`，价格需业务确认。
- 普通会员享受会员价购买精品 AI 课程 20 小时/单科；卓越会员一年内可无限次享受会员价购买精品 AI 课程。
- 普通会员受有效期和折扣课时双重限制：到期或课时用尽都会导致权益不可继续使用；到期后折扣课时清零。
- 卓越会员原则上按有效期判断权益，不按折扣课时用尽判断；是否仍保留后台折扣课时字段需确认。
- 普通会员可升级为卓越会员；卓越会员不能购买普通会员，原型提示“当前账号为卓越会员，无法购买普通会员权益”。
- 渠道商端可给家长开通/续费会员，每次开通/续费消耗 1 个会员名额，并为家长开通会员身份及充值 100 折扣课时。该规则同时属于渠道商名额闭环。
- 后台 `会员管理` 可为会员赠送折扣课时，确认后总折扣课时和剩余课时同步增加。
- 会员权益影响课程售卖闭环中的会员价、折扣课时计算和订单金额。

## 6. PC/移动端差异

PC 端：

- PC 后台负责会员权益包查询、充值记录、消耗明细、赠送折扣课时和会员充值订单查看。
- `会员管理` 同时支撑会员权益闭环和渠道商名额闭环。

移动端：

- 小程序负责开通、续费、升级、查看权益和查看折扣课时消费。
- `我的会员权益` 不承担后台配置，只展示用户侧权益结果。

历史/暂废：

- `我的会员权益（2026_6_8）` 等带日期页面仅作为版本差异参考。
- 历史支付页未确认前不进入当前主链路。

## 8. 页面与跳转逻辑

```text
小程序我的 -> 立即开通/查看权益
-> 我的会员权益 -> 续费/升级
-> 会员充值订单记录充值结果
-> 会员管理生成或更新权益包
-> 课程购买时读取会员类型和折扣课时
-> 购课消耗折扣课时
-> 我的会员权益和会员管理同步剩余课时
-> 到期或课时用尽后权益失效
```

渠道商支线：

```text
渠道商端给家长开通/续费会员
-> 消耗渠道商名额
-> 会员管理生成/更新家长会员权益包
-> 我的会员权益展示生效结果
```

## 9. 待确认问题

- 普通会员 `499元/年`、卓越会员 `1000元/年` 是否为正式价格。
- 普通会员续费后折扣课时是恢复固定课时、叠加剩余课时，还是覆盖原剩余课时。
- 卓越会员是否仍需要后台赠送折扣课时，还是仅按有效期判断。
- 卓越会员购买普通会员时是否正式采用原型阻断文案。
- 渠道商开通会员时开通普通会员、卓越会员，还是固定权益包。
- 共享账号字段来源和共享规则。

## 10. 风险与依赖

- 会员权益直接影响课程售卖价格和订单金额，价格未确认会影响售卖闭环。
- 渠道商开通会员依赖渠道商名额闭环，名额规则不清会影响会员来源追踪。
- 折扣课时消耗依赖购课订单和排课课时口径，需避免课时定义混用。
