# 需求分析：创客中心分销需求

## 背景

创客中心是分销员在用户端查看团队、收益和提现状态的入口。当前分销、佣金、提现已进入 MVP，但原型评审中发现“页面画面、评审说明、后台内容”容易混在一起，影响需求判断。本稿只分析创客中心需求，作为后续原型设计输入，不绘制原型，不定义接口、数据库或组件实现。

## 目标

明确创客中心的业务规则、字段来源、按钮动作、PC/移动端差异、页面跳转和待确认问题。重点约束：用户端创客中心默认首屏只展示用户真实可见内容；解释性文案可以同屏出现，但必须以说明标签或便签形式与产品 UI 明显区分。

## 关键结论

- 创客中心是用户端分销入口，主要服务分销员查看团队、收益和提现；PC 端后台需要提供支撑前端业务的配置、审核和团队管理页面。
- 平台内人人都是分销员，家长和老师均具备分销身份；但后台可以控制具体用户是否开通分销权限。未开通用户在个人中心仍需要有明确创客中心入口，并提示申请开通。
- 分销为二级分销，前端统一使用“一星团队、二星团队”命名；分销员可看到自己的一星团队，以及每个一星成员对应的全部二星团队。
- 当前贡献来源先覆盖注册、买课等；后续对接排课系统后，再展示上课等履约贡献。
- 创客中心主路径调整为：个人中心 → 创客中心入口展示 → 团队成员列表 → 提现中心 → 提现申请。
- 前端用户提交提现申请后，后台收到申请；财务根据申请在线下/系统外执行打款，打款成功后再回到系统内操作“打款成功”完成状态更新。
- 未开通分销权限的用户提交申请后展示等待审核状态；被拒绝后展示拒绝提示，并允许重新申请。
- 家长端和教师端个人中心的创客中心入口位置、入口文案一致；分销权限申请被拒绝时展示具体拒绝原因。
- 团队成员行点击后进入独立成员详情页；提现记录点击后进入提现详情页。
- 提现申请收款信息采用银行卡，并通过登录账号短信验证；银行卡采集开户银行、开户人姓名、银行卡号，不做实名校验；可提现金额为 0 时提现按钮置灰展示。
- PC 后台提现申请、分销比例配置、用户列表-团队列表由后台系统自定义权限人工配置的角色操作。
- 分销比例配置需要区分注册、买课、上课等不同来源，以及一星/二星不同层级比例，并支持生效时间、停用状态和历史版本记录。
- 成员贡献金额按系统配置的分成比例计算并展示，用于说明该成员为当前用户带来的分销金额。
- 解释性内容可以与原型同屏，但必须放入说明标签、黄色说明区或画布便签，不得混入真实页面标题、列表、按钮或业务状态。
- 提现时效不做承诺，用户端不能写成最终到账承诺；收款方式已确认为银行卡加登录账号短信验证，银行卡采集开户银行、开户人姓名、银行卡号且不做实名校验；金额展示按系统配置分成比例表达，不写死比例或默认金额。

## 1. 背景与范围

本次需求面向平台内所有用户，包括家长和老师。所有用户具备分销身份口径，但是否能看到和使用创客中心由后台分销权限控制。创客中心需要解决“团队看不清、收益来源不透明、提现状态不明确”的问题；PC 后台需要支撑分销权限开通、提现申请处理、分销比例配置、用户列表和团队列表查看。本次包含移动端个人中心入口展示、创客中心首页、团队成员列表、成员详情、提现中心、提现申请、提现详情，以及 PC 后台提现申请、分销比例配置、用户列表-团队列表模块；不包含自动打款、多级分销、跨品牌分销中台和接口设计。

## 5. PC/移动端差异

移动端：
- 本次创客中心需求主要面向移动端小程序，从个人中心进入。
- 移动端需要覆盖个人中心入口展示、申请开通/等待审核/被拒绝状态、创客中心首页、团队成员列表、成员详情、提现中心、提现申请、提现详情和提交/处理中状态。
- 后台分销权限控制不在移动端配置，但移动端需要体现权限结果：已开通用户可进入，未开通用户仍展示入口并提示申请开通。
- 页面空间有限，说明文案不能混入手机页面主体；原型中应放到手机画面外的说明标签或画布便签。

PC 端：
- 用户端创客中心 PC 端本次不作为主范围。
- PC 后台本次纳入支撑范围，需要覆盖提现申请处理、分销比例配置、用户列表-团队列表模块。
- PC 后台用于支撑移动端入口权限、团队关系、分销比例、提现处理，不应混入移动端创客中心首屏。

端差异待确认：
- 暂无端差异待确认；家长端和教师端个人中心入口位置、入口文案一致。
- PC 后台各页面由后台系统自定义权限人工配置的角色操作，具体角色名称不在本需求分析中写死。

## 6. 页面与跳转逻辑

本次创客中心原型建议包含以下用户端页面：

1. 个人中心入口展示
- 用于展示创客中心入口和当前用户是否可进入创客中心。
- 从个人中心进入或在个人中心中展示入口。
- 已开通分销权限时，点击入口进入创客中心首页。
- 未开通分销权限时，展示申请开通提示，不进入团队和收益页面。
- 申请提交后展示等待审核；审核拒绝后展示具体拒绝原因，并允许重新申请。

2. 创客中心首页
- 用于展示团队规模、今日新增、可提现、待结算和团队成员入口。
- 从个人中心入口展示进入。
- 点击团队成员入口进入团队成员列表。

3. 团队成员列表
- 用于展示一星团队、一星成员对应的全部二星团队、成员来源/行为记录和成员贡献金额。
- 从创客中心首页进入。
- 二星团队在一星成员下以树形分组或下钻方式展示。
- 点击成员行进入独立成员详情页。
- 点击提现入口进入提现中心。

4. 成员详情页
- 用于查看单个成员基础信息、所属一星/二星团队、贡献来源、贡献金额和关联团队信息。
- 从团队成员列表点击成员行进入。
- 点击返回回到团队成员列表。

5. 提现中心
- 用于展示可提现余额、收益记录、提现记录和申请入口。
- 从团队成员列表、成员详情页或创客中心首页提现入口进入。
- 点击“申请提现”进入提现申请。
- 可提现金额为 0 时，提现按钮置灰展示。
- 点击提现记录进入提现详情页。

6. 提现申请
- 用于填写提现金额和收款信息。
- 从提现中心进入。
- 收款信息采用银行卡，并通过登录账号短信验证码校验。
- 提交成功后展示已提交/处理中状态，并可返回提现中心查看记录。
- 后台收到申请后，财务系统外打款；打款成功后在系统内操作“打款成功”，用户端提现记录状态随之更新。

7. 提现详情页
- 用于查看单笔提现申请的金额、银行卡、申请时间、当前状态和处理结果。
- 从提现中心点击提现记录进入。
- 点击返回回到提现中心。

8. PC 后台-提现申请
- 用于查看和处理前端提交的提现申请。
- 财务可根据申请系统外执行打款，并在系统内标记打款成功。
- 驳回后用户端提现详情展示被拒绝状态。

9. PC 后台-分销比例配置
- 用于配置注册、买课、上课等不同来源，以及一星/二星不同层级的分销比例。
- 支持设置生效时间、启用/停用状态，并保留历史版本记录。
- 配置结果影响前端成员贡献金额和收益展示。

10. PC 后台-用户列表-团队列表
- 用于查看用户列表、用户分销权限状态、一星团队和二星团队关系。
- 可支撑开通申请审核、拒绝、重新开通，以及团队关系核查。

11. 原型说明标签区
- 不是业务页面，属于原型评审辅助区域。
- 与上述页面同屏展示，用于说明页面目的、入口/出口、字段规则、待确认项。
- 不参与用户真实操作链路。

### 重点说明

需要用说明标签展示：
- 页面入口/出口。
- 人人都是分销员，家长和老师都能拉新并获得分润。
- 后台可控制用户是否开通分销权限；未开通用户仍展示入口，并提示申请开通。
- 家长端和教师端个人中心入口位置、入口文案一致。
- 申请开通后展示等待审核；被拒绝后展示具体拒绝原因并允许重新申请。
- 二级分销定义：前端统一叫一星团队、二星团队；当前用户可查看一星团队，以及每个一星成员对应的全部二星团队。
- 二星团队以一星成员下钻、树形分组方式展示。
- 当前展示注册、买课等分润触发范围；上课贡献待后续排课系统对接后展示。
- 团队成员行点击后进入独立成员详情页。
- 成员贡献金额按系统配置的分成比例展示，不在用户端写死比例。
- 可提现、待结算、审核中等状态说明。
- 可提现金额为 0 时，提现按钮置灰展示。
- 提现申请采用银行卡 + 登录账号短信验证；银行卡采集开户银行、开户人姓名、银行卡号，不做实名校验。
- 提现记录点击后进入提现详情页。
- 提现申请由用户提交，后台收到后由财务系统外打款，打款成功后系统内标记成功；提现时效不做承诺。
- PC 后台需要支撑提现申请、分销比例配置、用户列表-团队列表模块，并由后台系统自定义权限人工配置的角色操作。
- 分销比例配置需要区分注册、买课、上课等不同来源，以及一星/二星不同层级比例，并支持生效时间、停用状态和历史版本记录。

不应放入产品 UI 主体：
- 移动端创客中心内的后台管理入口。
- 流程图入口。
- 原型评审按钮。
- 大段说明文字。
- 未确认规则的承诺性文案。
