## 一、排序方法论

### 1.1 三维度排序模型

本排序采用 **KANO × 商业价值 × 实现成本** 三维度模型：

| 维度 | 评估标准 | 权重 |
|------|---------|------|
| **KANO 分类** | 必备(M) → 期望(O) → 兴奋(A) → 无差异(I) | 40% |
| **商业价值** | 差异化壁垒 + 收入贡献 + 用户留存 × 市场机会 | 40% |
| **实现成本** | 低(1) → 中(2) → 高(3)，成本越低优先级越高 | 20% |

### 1.2 综合评分公式

```
优先级分值 = KANO分 × 0.4 + 商业价值分 × 0.4 + 成本反向分 × 0.2

KANO分: M=5, O=3, A=2, I=1
商业价值分: 5=极高, 4=高, 3=中, 2=低, 1=极低
成本反向分: 低成本=5, 中成本=3, 高成本=1
```

## 五、MVP 关键路径分析

### 5.1 领榜 C2C 核心路径

```
家长发布需求 → 需求进入大厅 → 教师浏览并付费接单 → 需求单锁定
→ 企微建群沟通 → 课后评价 → 需求完成/退单重新开放
```

**关键依赖**：LB-P-01 → LB-T-01 → LB-T-02（串联链路，任一缺失则闭环断裂）

### 5.2 优坐标 B2C 核心路径

```
家长浏览首页 → 查看课程/教师 → 购买课程 → 客服排课 → 教师上课
→ 课后评价
```

**关键依赖**：YZ-P-01 → YZ-P-02 → YZ-A-02（串联链路）

### 5.3 跨项目依赖

```
SH-01 微信登录（共用基础）

  └── 优坐标 B2C 核心路径

SH-05 数据隔离（共用基础）
```
