# 阶段规格输入/输出模板

> 版本：v1 | 创建时间：20260608
> 用途：K12 Codex 项目 SDAD 系统中，每个 Agent 必须使用此模板进行 I/O 校验

---

## 一、规格输入校验模板（阶段开始必用）

Agent 收到总控派发的任务包时，必须先在产物文件头部写入以下校验块：

```markdown
# [产物标题]

## 规格输入校验
> 校验时间：[YYYYMMDD-HHmm]
> 执行 Agent：[编号] [Agent名称]

| 序号 | 上游产物 | 文件路径 | 存在 | 版本 | 签字 | 结果 |
|-----|---------|---------|------|------|------|------|
| 1 | [上游产物名] | [完整路径] | ✅/❌ | vX | ✅/❌ | 通过/阻塞 |
| 2 | ... | ... | ... | ... | ... | ... |

**校验结论**：[全部通过 / 部分阻塞：xxx]

**阻塞处理（如有）**：
- [阻塞项]：[处理方式或已回报主控]
```

## 二、规格签字卡模板（阶段结束必用）

Agent 完成产物后，必须在产物文件末尾附加：

```markdown
---

## 规格签字卡
> 签字时间：[YYYYMMDD-HHmm]
> Agent：[编号] [Agent名称]

| 检查项 | 状态 |
|--------|------|
| 上游输入完整性 | ✅ 已消费 / ⚠️ 缺失xxx |
| 本阶段产物完整性 | ✅ 覆盖全部输出清单 / ⚠️ 缺少xxx |
| 未确认事项 | [清单] |
| 风险项 | [清单] |
| 越界声明 | 本产物不包含[下游阶段]的决策 |
| 签字 | ✅ 可交付给[下一阶段Agent编号] |

**下游须知：**
- [关键上下文1]
- [关键上下文2]
```

## 四、总控派发任务包模板

总控调度 Agent 向子 Agent 派发任务时，统一使用以下格式：

```markdown
# 任务包：[Agent编号] [Agent名称]

## 任务来源
- 用户原话：[原文]
- 主控理解：[理解]

## 任务目标
- [目标描述]

## 规格输入契约（必读）
| 编号 | 上游产物 | 文件路径 |
|-----|---------|---------|
| 1 | [产物名] | [路径] |

## 输出要求
- 产物目录：[目录]
- 文件名：[文件名]
- 必须覆盖：[输出清单]

## 已确认规则
- [规则1]
- [规则2]

## 禁止事项
- [禁止1]

## 验收标准
- [标准1]

## 下游交接
- 交付 Agent：[下一Agent编号]
- 关键上下文：[上下文]

---

**收到此任务包后，请先执行「规格输入校验」再开始工作。**
**完成后必须在产物末尾附加「规格签字卡」。**
```

---

## 五、主控校验清单（每阶段完成后）

| 检查项 | 标准 |
|--------|------|
| 文件写入正确目录 | 与 Agent 编号对应 |
| 文件命名符合规则 | `主题_内容_版本_时间` |
| 产物包含输入校验块 | 执行过输入校验 |
| 产物包含签字卡 | 已签字 |
| 产物覆盖输出清单 | 清单项目逐一检查 |
| 无越界内容 | 不包含下游阶段决策 |
| 无未确认假设 | 所有假设已标记 |
