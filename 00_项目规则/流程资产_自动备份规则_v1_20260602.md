# 流程资产自动备份规则

## 背景

本规则用于确保 k12 项目多助手协作流程资产可以随时还原、复用和部署。

## 自动备份策略

采用双层策略：

- 主控变更后立即备份：项目主控助手每次修改流程资产后，必须主动运行自动备份脚本。
- Codex 自动化兜底检查：每小时检查一次流程资产是否变化，如有变化则更新模板仓库并提交 Git 记录。

## 自动备份范围

自动备份只覆盖流程资产：

- AGENTS.md
- 00_项目规则/
- 13_任务卡片/

## 不自动备份内容

- 用户业务材料
- work/ 临时材料
- 会议记录正文
- 阶段业务产物
- 高保真 HTML 原型
- PRD
- 评审记录

## 模板仓库位置

- outputs/流程资产_模板仓库_v1_20260602/

## 自动备份脚本

- 00_项目规则/流程资产_自动备份脚本_v1_20260602.sh

## 执行规则

脚本执行时：

- 如果模板仓库不存在，则创建。
- 如果模板仓库不是 Git 仓库，则初始化 Git。
- 同步 `AGENTS.md`、`00_项目规则/`、`13_任务卡片/`。
- 更新备份说明、还原部署说明、目录初始化脚本和流程资产清单。
- 如果检测到变化，则提交 Git。
- 如果没有变化，则不生成空提交。

## 提交信息格式

自动提交信息格式：

```text
自动备份流程资产 YYYYMMDD HH:MM
```

## 主控要求

项目主控助手每次修改以下文件后，必须主动运行自动备份脚本：

- AGENTS.md
- 00_项目规则/ 下的任意流程规则或模板
- 13_任务卡片/ 下的任意任务卡片

## 风险与依赖

- Codex 自动化不是实时文件监听器，因此自动化只作为兜底检查。
- 变更后立即备份由项目主控助手负责执行。

## 下一步动作

- 每次流程资产变更后运行自动备份脚本。
- 定期检查 `outputs/流程资产_模板仓库_v1_20260602/` 的 Git 提交记录。
