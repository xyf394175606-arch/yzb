# draw.io 流程图兼容规则

## 使用目的

本文件用于约束业务流程助手输出 draw.io 兼容的流程图资料。

当前已安装 drawio-skill，但 Codex 需要重启后才能在会话中识别新 skill。

drawio-skill 还依赖本机 draw.io 桌面 CLI。如果 drawio-skill 或 draw.io CLI 不可用，则先输出可被后续 draw.io 工具或人工建图使用的结构化内容，不直接声称已经生成 .drawio 文件。

## 输出原则

每张图必须同时提供：

- 图名称
- 图类型
- Mermaid 代码
- 节点清单
- 连线清单
- 泳道/分组说明
- draw.io 建图建议

## 图类型

业务流程助手必须输出以下图：

- 业务动作流程图
- 角色泳道图
- 状态流转图
- 异常流程图
- 权限边界图

## 节点清单要求

节点清单必须包含：

| 字段 | 说明 |
| --- | --- |
| 节点ID | 用于图内引用，建议使用简短英文或编号 |
| 节点名称 | 节点在图中显示的名称 |
| 节点类型 | 开始、动作、判断、状态、异常、结束、角色、系统 |
| 所属泳道 | 如果是泳道图，必须填写 |
| 说明 | 节点含义 |

## 连线清单要求

连线清单必须包含：

| 字段 | 说明 |
| --- | --- |
| 起点 | 连线起点节点ID |
| 终点 | 连线终点节点ID |
| 条件 | 分支条件或触发条件 |
| 说明 | 连线含义 |

## 泳道/分组说明要求

泳道/分组说明必须包含：

| 字段 | 说明 |
| --- | --- |
| 分组名称 | 角色、系统或业务阶段 |
| 分组类型 | 角色、系统、阶段、权限域 |
| 包含节点 | 当前分组下的节点ID |
| 说明 | 分组含义 |

## draw.io 建图建议

每张图必须给出 draw.io 建图建议，包括：

- 建议图形类型
- 建议泳道或分组
- 判断节点样式
- 异常节点样式
- 状态节点样式
- 颜色或标注建议

## 后续升级方向

如果 drawio-skill 和 draw.io CLI 均可用，可以基于本文件要求的结构化信息生成：

- .drawio
- .xml
- mxfile

## 禁止事项

- 不得声称已经生成 .drawio 文件。
- 不得编造 drawio skill 调用结果。
- 不得只给 Mermaid 而缺少节点清单和连线清单。
- 不得只给文字说明而缺少图结构。
