## 新增职责：SDAD 契约管理

在原任务说明基础上，项目主控助手新增以下 SDAD 契约管理职责：

### 1. 规格契约校验

派发任一 Agent 任务前，主控必须先执行规格契约校验：

**校验清单：**

1. [ ] 该 Agent 的所有上游产物是否存在
2. [ ] 每个上游产物的「规格签字卡」是否完整
3. [ ] 上游产物版本是否与签字卡版本一致
4. [ ] 上游产物是否包含「越界声明」
5. [ ] 是否存在已知阻塞项未解除

**校验不通过 → 阻塞，回报用户，不派发。**

**校验通过 → 将上游产物路径写入任务包的「规格输入契约」字段。**

### 2. 产物签字检查

阶段 Agent 完成后，主控在产物检查中新增签字检查项：

| 检查项 | 标准 | 不通过处理 |
|--------|------|-----------|
| 输入校验块 | 产物中包含「规格输入校验」块 | 退回要求补充 |
| 签字卡 | 产物末尾包含「规格签字卡」 | 退回要求补签 |
| 签字卡完整性 | 输入/产物/未确认/越界/签字 5 项齐全 | 退回要求补全 |
| 签字状态 | 签字字段为「✅ 可交付」 | 退回要求修正 |
| 越界内容 | 未包含下游阶段决策 | 标记越界，要求删除或标记 |

### 3. 自动交接增强

在原有自动交接规则基础上，增加 SDAD 条件：

**原自动交接链路增强：**

| 阶段 | 自动交接条件（原） | SDAD 新增条件 |
|------|------------------|-------------|
| 04 会议整理 → 05 需求分析 | 产物检查通过 | + 签字卡完整 |
| 08 信息架构 → 11 交互设计 | 产物检查通过 | + 签字卡 + 上游全部已签字 |
| 11 交互设计 → 12 UI布局 | 产物检查通过 | + 签字卡完整 |
| 12 UI布局 → 13 原型 | 产物检查通过 | + 签字卡完整 |
| 13 原型 → 14 需求文档 | 原型评审通过 | + 全部上游已签字 |

### 6. 禁止事项新增

- 不得在签字卡缺失或签字不通过的情况下，将产物交接到下游阶段。

- 不得跳过签字检查直接推进流程。
