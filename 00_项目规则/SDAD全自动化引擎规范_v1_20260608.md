## 一、引擎定位

SDAD 全自动化引擎是 00 总控调度 Agent 的执行内核，负责将 16 个 Agent 按依赖关系编排为自动推进的流水线，实现「触发即运行，运行即交付，交付即签字」的无人值守工作模式。

## 二、核心机制

### 2.1 触发条件

| 事件类型 | 触发条件 | 动作 |
|---------|---------|------|
| 流水线启动 | 用户发出「开始」「继续」「跑完」等执行指令 | 启动引擎，按 DAG 从未完成的第一个 Agent 开始 |
| 阶段自动衔接 | 上游 Agent 产物签字卡状态 = ✅ | 自动启动下游 Agent |
| 异常熔断 | 产物签字卡状态 = ⚠️ 且重试 ≥ 2 次 | 暂停流水线，报告异常，等待人工干预 |
| 全局超时 | 单 Agent 执行时间 > 10 分钟 | 标记超时，跳过该 Agent，记录异常 |
| 流水线完成 | Agent 16 备份签字完成 | 生成执行报告，自动退出 |

### 2.2 完成标准

每个 Agent 必须同时满足以下 3 项方可标记为 ✅：

1. **产物存在**：输出文件已写入指定目录，文件非空
2. **签字卡完整**：产物末尾包含标准签字卡，所有字段已填写
3. **输入已消费**：签字卡中「输入消费确认」逐条标注已读取

### 2.3 状态检测与自动衔接

```python
# 伪代码
def auto_pipeline():
    for agent_id in [10, 11, 12, 13, 14, 15, 16]:
        # Step 1: 检查上游依赖
        deps = get_dependencies(agent_id)
        if any(dep.status != COMPLETED for dep in deps):
            log(f"Agent {agent_id} 阻塞，等待上游")
            break

        # Step 2: 执行 Agent
        start_time = now()
        try:
            result = execute_agent(agent_id, timeout=600_000)  # 10分钟
        except TimeoutError:
            mark_status(agent_id, TIMEOUT)
            log(f"Agent {agent_id} 超时，跳过")
            continue

        # Step 3: 验证完成标准
        if verify_completion(agent_id):
            mark_status(agent_id, COMPLETED)
            update_ledger(agent_id)
            log(f"Agent {agent_id} 完成 ✓")
        else:
            retry_count = get_retry_count(agent_id)
            if retry_count < 2:
                mark_status(agent_id, RETRY)
                retry_agent(agent_id)
            else:
                mark_status(agent_id, FAILED)
                log(f"Agent {agent_id} 重试 2 次仍失败，熔断")
                break

    # Step 4: 全部完成，生成报告
    if all_completed():
        generate_execution_report()
```

### 2.4 全局超时与异常熔断

| 级别 | 条件 | 动作 | 恢复方式 |
|------|------|------|---------|
| **警告** | 单 Agent 耗时 > 5 分钟 | 日志记录，继续执行 | 自动恢复 |
| **超时** | 单 Agent 耗时 > 10 分钟 | 标记超时，跳过该 Agent | 人工重试或跳过 |
| **熔断** | 连续 2 个 Agent 失败 | 暂停整个流水线 | 人工介入诊断后恢复 |
| **致命** | 核心依赖缺失（如 SRS 文件不存在） | 立即停止 | 人工修复后重新启动 |

### 2.5 进度追踪

| 追踪维度 | 说明 |
|---------|------|
| **Agent 进度** | ⬜/🔄/✅/⚠️/🚫 五态流转 |
| **Sprint 进度** | Sprint 1(01-03) / Sprint 2(04-06) / Sprint 3(07-09) / Sprint 4(10-12) / Sprint 5(13) / Sprint 6(14) / Sprint 7(15-16) |
| **文件产出** | 每完成一个 Agent，产物路径写入台账 |
| **时间记录** | 每个 Agent 的开始时间、完成时间、耗时 |
| **异常日志** | 超时、重试、熔断、跳过的详细记录 |

### 2.6 执行报告

流水线完成后自动生成，包含：

1. **执行摘要**：总耗时、成功率、异常数
2. **Agent 逐项明细**：编号/名称/状态/耗时/产物路径/签字状态
3. **异常清单**：超时/重试/熔断/跳过的详细记录
4. **产物清单**：全部产出文件的路径和大小
5. **待确认问题**：所有 Agent 产出的待确认项汇总
6. **下游建议**：进入开发阶段的准备度评估

## 四、重试与恢复策略

| 场景 | 策略 | 最大重试次数 |
|------|------|------------|
| 文件写入失败 | 重试写入，间隔 3 秒 | 2 |
| 产物内容为空 | 重新执行 Agent | 2 |
| 签字卡缺失 | 补写签字卡 | 1 |
| 上下文溢出 | 精简输入，保留核心规格 | 1 |
| 依赖文件不存在 | 报告异常，不重试 | 0 |
