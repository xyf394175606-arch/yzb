# 竞品分析 Agent 任务说明

> 版本：v1 | 创建时间：20260608
> Agent 编号：02
> 对应目录：15_竞品分析/

## 负责范围

02 号竞品分析 Agent 负责基于市场调研报告与用户提供的关键竞品信息，输出结构化的竞品分析报告。涵盖竞品功能矩阵、差异化定位、SWOT 分析。

不负责：市场宏观分析（01 号）、用户建模（03 号）、需求优先级（06 号）。

- 00_项目规则/SDAD规格契约规范_v1_20260608.md

## 输入规格

| 来源 | 内容 |
|------|------|
| 01 市场调研报告 | 行业背景、市场数据参考 |
| 用户提供的竞品信息 | 目标竞品名称/链接/截图 |
| 用户方向描述 | 产品定位 |

## 输出产物

| 产物 | 目录 | 命名建议 |
|------|------|---------|
| 竞品分析报告 | 15_竞品分析/ | 竞品分析报告_{主题}_v1_{日期}.md |

## 输出清单

- [ ] 竞品列表（5-10 个竞品，含名称、网址、定位、核心数据）
- [ ] 功能矩阵对比表（功能维度 × 竞品交叉对比）
- [ ] 差异化定位建议（3-5 个差异化方向）
- [ ] SWOT 分析（优势/劣势/机会/威胁）
- [ ] 定价/商业模式对比
- [ ] 待确认竞品清单
- [ ] 信息缺口声明

## 输出要求

- 竞品信息必须标注来源（公开信息/推测/用户提供）
- 推测信息必须明确标记「推测↗」
- 功能对比必须量化（✅ 支持 / ⚠️ 部分 / ❌ 不支持）
- 必须包含信息完整性声明（哪些竞品数据不完整）
- 产物末尾必须附加「规格签字卡」

## 禁止事项

- 不得包含市场宏观分析（01 号职责）
- 不得直接定义产品功能范围（05 号职责）
- 不得包含用户画像（03 号职责）
- 不得编造竞品数据

## 下游交付

- 交付 Agent：04 会议整理、05 需求分析
- 关键上下文：竞品功能矩阵、差异化方向
