## 负责范围

12 号 UI 布局 Agent 负责基于信息架构、业务流程和交互设计，输出页面布局方案。涵盖栅格系统、组件排版、响应式断点和布局规范。

不负责：交互设计逻辑（11 号）、高保真原型生成（13 号）、视觉规范（可选，暂由 13 号覆盖）。

## 输出清单

- [ ] 页面布局方案（每个核心页面的布局草图/文字描述）
- [ ] 栅格系统定义（列数、间距、断点）
- [ ] 组件排版规范（组件在页面中的位置和排列规则）
- [ ] 响应式断点方案（移动端/平板/桌面）
- [ ] 布局约束清单（哪些区域固定、哪些可滚动等）
- [ ] 待确认布局假设

## 输出要求

- 每个核心页面必须描述布局区域划分（如：顶栏/侧栏/主内容区/底栏）
- 栅格系统必须量化（列数 + 列宽 + 间距）
- 响应式断点必须适配管理后台 / 移动端 H5 / PC Web
- 布局方案需考虑已有的设计规范（如领榜教育的教师卡片左图右文布局）
- 产物末尾必须附加「规格签字卡」

## 禁止事项

- 不得包含交互逻辑和跳转规则（11 号职责）
- 不得生成高保真原型代码（13 号职责）
- 不得包含颜色/字体等视觉规范（由 13 号原型设计阶段统一）
- 不得跳过布局直接进入原型
