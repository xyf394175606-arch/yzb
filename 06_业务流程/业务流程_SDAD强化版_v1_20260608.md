## 二、新增业务流程设计

### 2.1 GAP-01: 教师资质认证审核流程

**适用项目**：领榜教育 + 优坐标（两套独立审核）

**主流程**：

```
教师提交资质（身份证/学历证/资格证/教学经历）
    → 系统创建审核工单（状态：待审核）
    → 后台管理员审核
        → 审核通过 → 教师认证标识生效（认证教师/星级教员）
                    → 接单权重提升（信用体系联动）
        → 审核驳回 → 通知教师补充材料
                    → 教师重新提交（限3次）
                    → 3次驳回后冻结接单权限30天
```

**异常流程**：
- 资质伪造：管理员标记「疑似伪造」，冻结接单权限，通知客服跟进
- 资质过期：系统定时检查，到期前30天提醒，到期后取消认证标识

**权限边界**：
- 教师：只能提交和查看自己的资质审核状态
- 后台管理员：可审核、驳回、标记疑似伪造
- 客服：可查看审核状态，不可审核

### 2.2 GAP-02: 教师信用体系流程

**适用项目**：领榜教育（优坐标 V2）

**信用评分模型**：

| 维度 | 权重 | 加分项 | 扣分项 |
|------|------|--------|--------|
| 完单率 | 30% | 完单+2/单 | 退单-5/单，超时-3/单 |
| 评价分 | 25% | 5星+1/单 | 3星以下-2/单 |
| 认证等级 | 20% | 认证教师+10，星级教员+20 | 认证取消-20 |
| 响应速度 | 15% | 24h内接单+1/单 | 48h未响应-1/单 |
| 投诉率 | 10% | 无投诉+5/月 | 被投诉-5/次 |

**信用等级划分**：

| 等级 | 分值区间 | 接单权限 | 展示标识 |
|------|---------|---------|---------|
| S 级 | 90-100 | 可接所有需求，优先展示 | 🌟 星级教员 |
| A 级 | 70-89 | 可接所有需求 | ✅ 认证教师 |
| B 级 | 50-69 | 可接部分需求（排除高价值需求） | 📋 普通教师 |
| C 级 | 0-49 | 接单受限，需重新认证 | ⚠️ 观察期 |

**信用评分联动流程**：

```
订单完成 → 评价提交 → 系统自动计算信用分变动
    → 分值跨等级边界 → 自动升降级
        → 升级：立即生效，接单权限扩展
        → 降级：通知教师，7天内可申诉
            → 申诉通过 → 恢复等级
            → 申诉驳回 → 维持降级，30天后可重新评定
```

### 2.3 GAP-03/06: 投诉提交流程

**适用项目**：领榜教育 + 优坐标（两套独立投诉）

**主流程**：

```
用户（家长/教师）提交投诉
    → 选择投诉类型（服务质量/费用争议/教师行为/系统问题）
    → 填写投诉内容 + 上传证据（图片/视频）
    → 系统生成投诉工单（状态：待处理）
    → 分配处理人
        → 优先分配对应项目客服
        → 涉及费用争议 → 转财务
        → 涉及教师行为 → 转管理员
    → 客服/管理员处理
        → 联系双方了解情况
        → 提出处理方案
        → 方案确认
    → 通知投诉人处理结果
    → 投诉人确认/不满意
        → 确认 → 工单关闭
        → 不满意 → 升级处理（管理员二次审核）
```

**处理时效承诺**：

| 投诉类型 | 首次响应 | 处理完成 |
|---------|---------|---------|
| 服务质量 | 2h | 48h |
| 费用争议 | 4h | 72h |
| 教师行为 | 1h | 24h |
| 系统问题 | 4h | 48h |

**与信用体系联动**：
- 投诉成立且涉及教师行为：教师信用分 -5~-10（根据严重程度）
- 投诉不成立：不影响信用分

### 2.4 GAP-04: 试课体验流程

**适用项目**：领榜教育（V2 版本）

**主流程**：

```
家长申请试课
    → 选择科目/年级/时间偏好
    → 系统匹配可用教师（信用B级以上）
    → 教师确认接试课单
    → 试课进行（1节，30-60分钟）
    → 试课结束
        → 家长满意 → 转正式需求单（优先匹配试课教师）
        → 家长不满意 → 可重新申请试课（每月限2次）
        → 试课教师获得试课补贴（平台承担）
```

**注**：试课体验为 V2 版本需求，当前 MVP 不纳入。

### 2.5 GAP-05/08: 教师课时费提现流程

**适用项目**：领榜教育 + 优坐标

**主流程**：

```
教师发起提现申请
    → 选择提现金额（可用余额范围内）
    → 选择提现方式（银行卡/微信）
    → 系统创建提现工单（状态：待审核）
    → 财务审核
        → 审核通过 → 状态变更为「待打款」
        → 审核驳回 → 通知教师原因
    → 系统外打款
    → 财务确认到账 → 状态变更为「已完成」
    → 教师收到到账通知
```

**异常流程**：
- 提现金额异常（单笔超5000）：需管理员复核
- 打款失败：财务标记失败原因，重新发起打款
- 余额不足：拒绝提现，通知教师查看明细

**业务规则**：
- 结算周期：T+3（申请后3个工作日到账）
- 最低提现金额：100元
- 手续费：平台承担（MVP阶段）

**与分销提现的区别**：

| 维度 | 教师课时费提现 | 分销佣金提现 |
|------|-------------|------------|
| 触发条件 | 订单完成后课时费到账 | 佣金转为可结算后 |
| 金额来源 | 教师上课所得 | 分销推广所得 |
| 审核角色 | 财务 | 财务 |
| 打款方式 | 相同 | 相同 |

### 2.6 GAP-09: 系统消息通知流程

**适用项目**：两项目共用机制

**通知类型与渠道**：

| 通知类型 | 触发事件 | 渠道 | 接收角色 |
|---------|---------|------|---------|
| 需求通知 | 新需求发布/需求被接单 | 小程序站内 + 微信服务号 | 教师/家长 |
| 订单通知 | 接单成功/退单/退款 | 小程序站内 + 微信服务号 | 教师/家长 |
| 审核通知 | 资质审核结果/提现审核结果 | 小程序站内 | 教师 |
| 信用通知 | 信用等级变动 | 小程序站内 | 教师 |
| 投诉通知 | 投诉受理/处理结果 | 小程序站内 + 微信服务号 | 投诉人 |
| 分销通知 | 佣金到账/提现进度 | 小程序站内 | 分销员 |
| 系统通知 | 系统维护/功能更新 | 小程序站内 + 微信服务号 | 所有用户 |

**消息流程**：

```
业务事件触发 → 消息引擎生成消息
    → 按通知类型匹配渠道
    → 写入用户消息列表（站内信）
    → 推送微信服务号模板消息（需用户授权）
    → 用户查看 → 消息标记已读
    → 未读消息24h后二次提醒（仅重要消息）
```

```
    → 系统返回日志列表
    → 可导出Excel
    → 涉及敏感操作（财务/权限）需管理员二次确认查看
```

## 三、现有流程冲突修正

### 3.1 CF-02: 退单对教师信用评分的影响

在现有领榜 C2C 流程「老师退单后需求单自动重新开放」的基础上，补充信用联动：

```
老师退单
    → 需求单状态变为「重新开放」（现有流程不变）
    → 【新增】教师信用分扣减5分
    → 【新增】如果退单原因标记为「教师主动放弃」
        → 信用分额外扣减3分
        → 已付接单费用不予退还
    → 【新增】如果退单原因标记为「家长需求变更」
        → 不额外扣减
        → 已付接单费用退还50%（客服人工审核）
```
