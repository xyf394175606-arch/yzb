# 高保真原型：创客中心分销说明

## 背景

本版本基于 `05_需求分析/需求分析_创客中心分销需求_v3_20260621.md` 输出，用于把创客中心分销需求转成可评审的 Axure 风格 HTML 原型。原型同时覆盖移动端创客中心链路和 PC 后台支撑页面。

v8 按新的 `axure-prototype-design` 规则重构：导航必须展示层级，但状态不进入左侧导航；互斥状态放在所属页面规则区下方，用状态预览和“全部状态”说明表达。

## 目标

- 用线框图表达移动端创客中心主链路、状态页和提现流程。
- 用 PC 后台页面表达提现申请、分销比例配置、用户列表-团队列表的支撑关系。
- 用红字说明局部字段、按钮和状态规则。
- 用黄色说明区集中表达页面级业务规则和版本变化。
- 用三级导航表达：端范围、业务模块、页面/详情页。
- 个人中心权限状态、可提现金额为 0 状态不进入左侧导航，归属到对应页面的规则区。
- 每个存在状态的页面都用“状态预览 + 全部状态”覆盖默认状态和互斥状态。

## 关键结论

- 移动端包含：个人中心入口、创客中心首页、团队成员列表、成员详情、提现中心、提现申请、提现详情。
- 移动端状态说明包含：申请开通、等待审核、拒绝后重申、可提现金额为 0。
- PC 后台包含：提现申请、分销比例配置、用户列表-团队列表。
- 分销统一使用“一星团队 / 二星团队”命名。
- 未开通分销权限用户可申请开通，等待审核；被拒绝后展示具体拒绝原因并可重新申请。
- 提现采用银行卡 + 登录账号短信验证，不做实名校验。
- 分销比例配置支持注册、买课、上课来源，一星/二星层级，生效时间、停用状态和历史版本记录。
- 左侧导航层级为：入口；移动端下分个人中心、创客中心、提现；PC 后台下分提现管理、分销配置、用户管理。
- 左侧导航不放状态；状态统一在所属页面规则区内说明。
