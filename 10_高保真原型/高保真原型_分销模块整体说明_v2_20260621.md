# 高保真原型：分销模块整体说明

## 背景

本版本基于 `05_需求分析/需求分析_分销模块整体需求_v2_20260621.md` 输出，用于验证新版 `requirement-analysis` 和新版 `axure-prototype-design` 的设计效果。v2 不覆盖 v1，重点检查“真实 UI 与说明分离”和“后台列表行操作必须使用业务弹窗”两类新增规则。

## 目标

- 用移动端线框表达个人中心、创客中心入口、创客中心首页、团队成员列表、成员详情页、提现中心、提现申请、提现详情页。
- 用 PC 后台线框表达后台提现申请、分销比例配置、用户列表-团队列表。
- 确保只有用户真实可见内容进入手机/PC 线框。
- 用红字说明局部字段、按钮、列表行和状态规则。
- 用黄色说明区承接页面规则、计算口径、内部规则、待确认项和全部状态。
- 用业务弹窗承接后台行操作，不使用通用说明弹窗或浏览器 `alert`。

## 关键结论

- 原型文件：`10_高保真原型/高保真原型_分销模块整体原型_v2_20260621.html`
- 需求分析输入：`05_需求分析/需求分析_分销模块整体需求_v2_20260621.md`
- 页面清单列顺序为：`端、页面、页面用途、页面归属`。
- 左侧导航按页面层级树组织，不包含等待审核、已拒绝、空状态、无权限、提交失败等状态。
- 移动端真实 UI 未加入后台管理入口、评审入口、流程说明或来源摘要模块。
- “注册、买课、上课是分润来源”作为计算口径放入黄色说明区，不作为首页 UI 模块。
- 后台提现申请、分销比例配置、用户列表-团队列表的关键行操作均使用业务弹窗。
- 提现时效不做承诺，用户端不展示固定到账时间。

## 新版 Skill 验证点

| 验证点 | v2 处理 |
| --- | --- |
| 只有用户可见内容进入线框 | 已按内容展示属性处理，计算口径和内部规则放黄色说明区 |
| 不使用通用解释按钮或浏览器提示 | 原型脚本未使用浏览器提示函数，后台行操作用业务弹窗 |
| 后台行操作展示当前行字段 | 弹窗带入申请人、联系方式、金额、收款信息、配置来源、权限状态等当前行字段 |
| 业务弹窗包含取消/确认结果 | 每个弹窗展示确认后结果和取消后结果 |
| 页面清单列顺序正确 | 使用 `端、页面、页面用途、页面归属` |
| 状态不进入导航 | 状态统一放页面黄色说明区 |
| 待确认项不写成已确认能力 | 打款流水号、关闭权限后历史收益展示、风控和退款冲正均保留待确认 |

## 待确认问题

- 标记打款成功是否需要填写系统外打款流水号。
- 关闭分销权限后，历史收益和提现记录在用户端的展示口径。
- 分销风控、退款/退单/课消撤销后的佣金冲正细则。

## 风险与依赖

- 上课贡献依赖后续排课系统对接，当前仅在说明区标注，不写成已上线能力。
- PC 后台操作角色由系统自定义权限人工配置，原型不写死固定岗位名称。
- 原型中的金额、姓名、手机号均为示例数据，不代表真实业务数值。
- 如后续新增页面或字段，应先回流需求分析，补齐内容展示属性和按钮/弹窗结果后再改原型。
