# 高保真原型说明：分销模块平面画布 v2

## 背景

用户要求调用 `interactive-canvas-prototype` 能力，将分销模块原型调整为平面画布形式，并且只保留两个主页面：分销前端页面和对应后台页面。本轮基于 `高保真原型_分销模块线框图原型_v1_20260621.html` 新建 v2，不覆盖旧版本。

## 目标

交付一个支持 `交互模式 / 画布模式` 的可交互 HTML 原型。交互模式用于查看两个主页面的可操作线框，画布模式用于平铺查看前端页与后台页的对应关系，并支持节点拖拽、画布缩放、便签编辑和导出。

## 关键结论

- v2 只保留两个业务主页面：`分销前端页面` 与 `分销后台页面`。
- 前端页整合创客中心、团队、收益、提现记录和提现申请，不再拆成多个主导航页面。
- 后台页整合分销总览、分销商、佣金、提现审核、规则配置和报表入口，不再拆成多个主导航页面。
- 画布模式只放两个页面节点：`distribution-frontend` 与 `distribution-backend`。
- 画布便签与交互标注共享语义，便签变更会同步到本地 annotation 数据。
- 未确认的财务规则继续显示为“待确认/按配置”，不写死优坐标默认佣金数值，也不承诺 T+3。

## 风险与依赖

- v2 是线框/画布评审原型，仍不是最终开发稿。
- 便签和节点位置依赖 localStorage；导出 HTML 可固化当前数据。
- 若后续并入主原型，需要再次检查主原型导航、标注和本地存储 key 冲突。
