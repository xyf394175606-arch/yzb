# 高保真原型：分销模块整体说明

## 背景

本版本按照新版 `axure-prototype-design` 规则重新设计分销模块原型。v3 不复用既有 v1/v2 原型结构、样式或交互实现，只使用 `05_需求分析/需求分析_分销模块整体需求_v2_20260621.md` 中的业务内容作为输入。

## 目标

- 用单文件 HTML 原型表达分销模块移动端与 PC 后台页面。
- 采用“左侧页面导航 + 中间真实线框 + 右侧黄色需求标签”的评审结构。
- 中间手机/PC 线框只放真实页面内容。
- 所有字段、按钮、列表、状态、页面、弹窗、待确认说明统一放右侧黄色标签。
- 后台列表行操作使用真实业务弹窗，不使用浏览器提示或通用解释弹窗。

## 关键结论

- 原型文件：`10_高保真原型/高保真原型_分销模块整体原型_v3_20260622.html`
- 覆盖移动端：个人中心、创客中心入口、创客中心首页、团队成员列表、成员详情页、提现中心、提现申请、提现详情页。
- 覆盖 PC 后台：后台提现申请、分销比例配置、用户列表-团队列表。
- 左侧导航按需求文件页面层级树组织，页面状态不进入导航。
- 中间线框区域未放说明性文字、规则卡片、状态预览或待确认说明。
- 注册、买课、上课作为计算口径放在右侧黄色标签，不生成移动端首页模块。
- 用户端不展示固定到账时间。
- 后台提现、配置启停、用户权限处理均通过当前行业务弹窗完成。

## 待确认问题

- 标记打款成功是否需要填写系统外打款流水号。
- 关闭用户分销权限后，历史收益和提现记录在用户端的展示口径。
- 风控、退款冲正和课消撤销后的佣金处理。

## 风险与依赖

- 上课贡献依赖后续排课系统对接，当前不能写成已上线能力。
- PC 后台操作角色由系统自定义权限配置，原型不写死固定角色名称。
- 原型中的姓名、手机号、金额为示例数据。
- 若后续补充分销模块正式 PRD，应以 PRD 重新校准页面字段和状态。

## 自检记录

- 文件命名符合 `文件主题_调整内容_版本号_创建时间`。
- HTML 内联脚本通过语法检查。
- 所有 `data-page` 跳转目标均有对应页面。
- 页面清单表头为 `端、页面、页面用途、页面归属`。
- 未使用浏览器提示函数或通用解释弹窗替代业务动作。
- 左侧导航未包含等待审核、已拒绝、空状态、无权限、提交失败等页面状态。
- 后台行操作弹窗包含当前行字段、取消按钮、确认按钮和确认后状态变化。
- 自动化截图环境如不可用，以 HTTP 访问、结构解析和脚本检查作为替代验证。
