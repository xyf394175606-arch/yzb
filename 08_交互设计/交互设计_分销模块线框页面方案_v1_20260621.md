# 页面方案：分销模块线框原型

## 背景

本方案基于分销模块专项需求分析和澄清清单整理，用于生成线框图级原型。当前分销模块已确认进入 MVP，但仍存在领榜分佣基数、优坐标默认佣金配置、冻结影响、退款冲正、风控和提现时效等待确认问题，因此本轮只整理页面结构、任务链路、字段取舍和前端依赖，不作为最终高保真视觉稿或最终 PRD。

## 目标

将分销模块从需求描述整理成可评审的线框页面结构，覆盖用户端创客中心和后台分销管理的核心路径，帮助后续业务流程、信息架构、交互设计、数据交互和高保真原型继续细化。

## 关键结论

- 原型分为两个表面：用户端创客中心和后台分销管理。
- 用户端主路径：进入创客中心 → 查看团队与收益 → 查看团队成员 → 进入提现中心 → 提交提现申请。
- 后台主路径：进入分销管理 → 查看分销商/佣金/提现 → 审核提现 → 查看或编辑佣金规则配置。
- 原型只体现线框结构、字段优先级、状态和权限差异，不写死待确认金额规则。
- 后台和流程信息不能出现在普通用户端默认首屏。

## 2. 核心任务链路

1. 分销员从“我的”进入创客中心。
2. 首屏先看团队网络、今日新增、可提现和待结算。
3. 分销员点击一星/二星团队查看成员列表或成员详情。
4. 分销员进入提现中心，查看收益记录和提现记录。
5. 分销员填写提现申请，系统提示人工审核和系统外打款。
6. 财务在后台进入提现审核列表，筛选待审核申请。
7. 财务打开审核抽屉，查看申请人、金额、来源、风险提示和历史记录。
8. 财务审核通过或拒绝；管理员可复核，客服无审核权限。
9. 运营或管理员查看佣金规则配置，确认领榜规则和优坐标按配置处理。

## 4. 页面关系

- 入口：用户端从“我的 - 创客中心”进入；后台从“分销管理”进入。
- 主路径：创客中心首页 → 团队成员详情 / 提现中心 → 提现申请；后台总览 → 提现审核 → 审核抽屉。
- 分支路径：后台总览 → 分销商管理 / 佣金记录 / 佣金规则配置 / 分销报表。
- 返回规则：用户端二级页面返回上一级；后台抽屉关闭后保留列表筛选条件。
- 状态保持：用户端保留当前 Tab；后台保留项目筛选、状态筛选和当前列表页。

## 5. 每页功能模块

### 创客中心首页

- 页面目标：让分销员快速知道团队规模、收益状态和下一步动作。
- 信息层级：团队网络优先，其次收益，再次成员预览和规则提示。
- 功能模块：团队统计、收益摘要、提现入口、一星成员预览、页面方案说明。
- 操作：查看提现、查看团队成员、显示/隐藏页面方案。
- 空状态/异常状态：暂无团队、收益加载失败、提现暂不可用。
- 前端依赖：团队数量、今日新增、可提现金额、待结算金额、最低提现配置、权限标记。

### 团队成员详情

- 页面目标：说明一星或二星成员是谁、从哪里来、做过什么、贡献如何。
- 信息层级：身份信息优先，行为事实其次，贡献金额弱化展示。
- 功能模块：基础资料、行为时间线、团队 Tab、二星成员列表。
- 操作：切换 Tab、查看二星成员、返回。
- 空状态/异常状态：无二星成员、行为记录为空、贡献待结算。
- 前端依赖：成员身份、脱敏手机号、来源页面、注册时间、购买/入驻事实、贡献汇总、下级数量。

### 提现中心

- 页面目标：让分销员理解收益和提现状态，并发起提现。
- 信息层级：可提现优先，收益记录和提现记录并列。
- 功能模块：余额摘要、收益记录、提现记录、申请提现入口。
- 操作：申请提现、查看记录。
- 空状态/异常状态：暂无收益、暂无提现记录、审核中不可重复提交。
- 前端依赖：可提现余额、待结算余额、收益状态、提现状态、审核状态、打款状态。

### 提现申请

- 页面目标：提交提现金额和收款信息，同时说明人工审核和系统外打款。
- 信息层级：金额输入优先，收款信息其次，规则提示固定展示。
- 功能模块：提现金额、姓名、收款账号/银行卡、开户信息、提交结果。
- 操作：提交、取消、返回提现中心。
- 空状态/异常状态：金额为空、金额超出可提现、收款信息缺失、提交失败。
- 前端依赖：余额、提现规则、收款字段配置、校验结果、提交状态。

### 后台分销总览

- 页面目标：给财务/管理员/运营一个进入分销管理的工作台。
- 信息层级：待处理优先，其次关键指标，再次模块入口。
- 功能模块：项目切换、待审核提现、规则异常、今日新增、模块导航。
- 操作：切换项目、进入提现审核、进入规则配置。
- 空状态/异常状态：无权限、指标加载失败。
- 前端依赖：项目权限、角色权限、待办数量、指标数据。

### 提现审核

- 页面目标：让财务完成审核，并让管理员复核，客服只读。
- 信息层级：申请人、金额、状态、风险提示、操作按钮。
- 功能模块：筛选区、列表、审核抽屉、审核结果。
- 操作：审核通过、拒绝、记录打款、关闭抽屉。
- 空状态/异常状态：无待审核、无权限、打款失败。
- 前端依赖：角色权限、申请状态、风险标记、历史记录、审核结果。

### 佣金规则配置

- 页面目标：区分领榜确认规则和优坐标按配置规则，支撑后续运营调参。
- 信息层级：项目先行，规则项分组，其次变更记录。
- 功能模块：项目切换、规则列表、配置表单、待确认提示。
- 操作：查看配置、保存草稿、查看变更记录。
- 空状态/异常状态：配置缺失、待财务确认、保存失败。
- 前端依赖：项目、规则类型、比例/金额、是否待确认、配置权限、变更记录。

## 7. 原型前检查

- 必须页面是否都在主路径中：是，用户端和后台主路径均覆盖。
- 可合并页面是否已经合并或说明理由：分销报表标为 Optional，未作为核心主路径。
- 字段是否完成 Show / Demote / Hide / Replace 取舍：已完成主要字段取舍。
- 每个主要字段是否都有用途和前端依赖：已标注到前端依赖层，不深入数据库或接口字段。
- 是否包含正常、空、受限/失败状态：已在页面模块和原型说明中覆盖。
- 是否需要调用设计优化 skill：本轮是线框图和管理系统，不使用营销型视觉规则；采用低保真、灰阶、结构优先。

## 8. 原型设计说明

- 布局策略：左侧为页面导航，主体分为移动端线框和后台线框两种表面。
- 交互策略：通过顶部分段控件切换用户端/后台，通过按钮模拟主要路径和状态。
- 样例数据策略：使用可体现正常、待确认、无权限和空状态的样例，不写死未确认金额规则。
- 标注策略：用编号标注解释关键模块、字段、权限和待确认点；支持显示/隐藏页面方案。

## 待确认问题

- 领榜分佣基数口径。
- 优坐标默认佣金配置和归因有效期。
- 分销商冻结后的具体影响。
- 退款、退单、课消撤销后的佣金冲正和已提现处理。
- 提现时效是否承诺 T+3。
- 创客中心是否在家长端和教师端完全共用。

## 风险与依赖

- 本方案不是最终信息架构和高保真原型，仍需后续阶段补齐流程、信息架构、数据交互。
- 金额和风控规则未确认前，原型只展示“按配置”和“待确认”。
- 若后续要并入主原型，应先完成业务流程与数据交互回流。
