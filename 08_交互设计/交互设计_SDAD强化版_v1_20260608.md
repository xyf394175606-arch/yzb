## 二、核心交互流程设计

### 2.1 教师资质认证交互流

```
[我的页] → 点击"资质认证"
    │
    ├── 状态=未提交 → [资质提交页]
    │     │
    │     ├── 填写姓名 → 上传身份证(必填) → 上传学历证(必填)
    │     │     → 上传资格证(选填) → 填写教学经历(选填) → 点击"提交"
    │     │         │
    │     │         ├── 提交成功 → Toast"已提交审核" → 跳转[审核状态页]
    │     │         └── 提交失败(参数缺失) → 对应字段红框+提示
    │     │
    │     └── 已有待审核记录(409) → Toast"您有待审核的认证，请耐心等待"
    │
    ├── 状态=待审核 → [审核状态页]
    │     └── 动画展示"审核中" + 预计1-3个工作日
    │
    ├── 状态=已通过 → [审核状态页]
    │     └── ✅ 认证标识展示 + 接单权重提升提示
    │
    ├── 状态=已驳回 → [审核状态页]
    │     └── ❌ 驳回原因列表 → [重新提交]按钮(≤3次)
    │
    └── 状态=已冻结 → [审核状态页]
          └── 🔒 冻结提示 + 解冻倒计时
```

### 2.2 信用体系交互流

```
[首页] → 信用分入口(卡片)
    │
    → [信用分展示页]
        ├── 仪表盘(0-100分, 弧形进度)
        ├── 等级标签: S(金)/A(银)/B(铜)/C(灰)
        ├── 接单权重提示文案
        │     S级: "您的信用优秀，接单优先推荐"
        │     A级: "您的信用良好，正常接单"
        │     B级: "信用待提升，部分优质需求可能无法抢"
        │     C级: "信用较低，建议减少退单和投诉"
        └── [查看明细] → [信用明细页]
              ├── 时间线列表
              │     ├── 🟢 +5 完成订单 #LB20260608001
              │     ├── 🔴 -10 收到投诉 #CP20260608003
              │     └── 🔴 -8 退单 #LB20260608005
              └── 点击订单号 → 跳转订单详情
```

### 2.3 投诉交互流

```
[订单详情] → 点击"投诉"
    │
    → [投诉提交页]
        ├── 选择投诉类型: 教学质量/排课问题/费用争议/其他
        ├── 填写描述(10-500字)
        ├── 上传证据图片(0-5张)
        └── 提交
              │
              ├── 成功 → [投诉进度页]
              │     ├── 时间线: 已提交 → 处理中 → 已解决
              │     ├── 处理人信息展示
              │     └── 超时48h → 红色预警 + "升级处理中"
              │
              └── 重复投诉 → Toast"该订单已存在同类型投诉"
```

### 2.4 提现交互流

```
[我的] → 点击"提现"
    │
    → [提现申请页]
        ├── 展示可提现余额(大字)
        ├── 输入提现金额(≥100元)
        ├── 选择收款银行账户
        ├── 展示手续费: ¥0.00
        ├── 展示预计到账: T+3工作日
        └── 点击"申请提现"
              │
              ├── 成功 → [提现记录页]
              │     └── 最新记录显示"待审核"
              │
              ├── 余额不足 → 提示"可提现余额不足"
              │
              └── 有待审核提现 → Toast"您有待审核的提现申请"
```

## 三、全局交互规则

### 3.1 加载状态

| 场景 | 交互 |
|------|------|
| 首次加载 | 骨架屏(列表页) / 加载动画(详情页) |
| 下拉刷新 | 下拉刷新组件 + Toast"已刷新" |
| 分页加载 | 触底加载 + loading 指示器 |
| 提交操作 | 按钮loading + 禁止重复点击 |

### 3.2 空状态

| 场景 | 交互 |
|------|------|
| 列表无数据 | 居中插图 + "暂无内容" + 引导操作按钮 |
| 搜索无结果 | "未找到相关内容" + 修改搜索条件提示 |
| 信用无记录 | "信用档案建立中" + 说明文案 |

### 3.3 错误状态

| 场景 | 交互 |
|------|------|
| 网络异常 | 全屏错误页 + "点击重试" |
| 接口错误 | Toast"操作失败，请稍后重试" |
| 表单校验失败 | 字段下方红色提示文案 |
| 权限不足 | Toast"您暂无此权限" + 返回 |

### 3.4 反馈机制

| 类型 | 交互 |
|------|------|
| 操作成功 | Toast(1.5s自动消失) |
| 操作确认 | 底部弹窗(确认/取消) |
| 删除/冻结 | 二次确认弹窗(红色警告) |
| 状态变更 | 状态标签颜色变化 + 轻振动反馈 |
