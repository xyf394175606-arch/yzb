## 一、产品概述

### 1.1 产品定位

K12 教育业务管理系统，包含两个独立业务平台：

| 平台 | 模式 | 核心价值 | 目标用户 |
|------|------|---------|---------|
| **领榜教育** | C2C 家教撮合 | 家长发布需求→教师付费接单→企微监管→闭环交付 | 家长+教师+客服+管理员 |
| **优坐标** | B2C 产品+排课 | 产品/会员购买→客服约课→教师分配→上课安排 | 家长+教师+客服+管理员 |

### 1.2 系统构成

| 端 | 数量 | 技术形态 |
|----|------|---------|
| 领榜家长端小程序 | 1 | 微信小程序 |
| 领榜教师端小程序 | 1 | 微信小程序 |
| 优坐标家长端小程序 | 1 | 微信小程序 |
| 优坐标教师端小程序 | 1 | 微信小程序 |
| 领榜 SaaS 后台 | 1 | Web SPA |
| 优坐标 SaaS 后台 | 1 | Web SPA |

### 1.3 技术架构

| 层级 | 技术 |
|------|------|
| 前端 | 微信小程序 + Vue3(Web) |
| 后端 | Java Spring Boot |
| 数据库 | MySQL(双租户物理隔离) + Redis |
| 消息 | RabbitMQ |
| 文件 | 阿里云 OSS |
| 部署 | 阿里云 ECS(MVP) → K8s(V5+) |

## 二、MVP 功能清单（25 条需求）

### 2.1 领榜教育 C2C 功能

| 编号 | 功能 | 优先级 | 描述 | 验收标准 |
|------|------|--------|------|---------|
| LB-P-01 | 发布需求 | P0 | 家长发布家教需求单，含科目/年级/频率/课时费/地址 | 表单提交成功→状态变更为"抢单中" |
| LB-P-02 | 查看我的需求 | P0 | 家长查看需求列表和详情，含状态跟踪 | 列表按状态分Tab，详情含生命周期时间线 |
| LB-P-03 | 教师付费接单 | P0 | 教师付费购买接单权，3倍课时费 | 付费成功→需求锁定→教师可联系家长 |
| LB-P-04 | 企微监管 | P0 | 后台记录企微沟通状态 | 状态：未建群/监管中/已完成/异常 |
| LB-P-05 | 未成交重新开放 | P0 | 退单/未成交后需求单自动重新开放 | 重新开放后教师可再次抢 |
| LB-P-06 | 会员购买 | P1 | 1年¥300/3年¥500，免费推荐老师 | 会员可发布需求后免费获得推荐 |
| LB-P-07 | 分销体系 | P1 | 二级分销，代理40%/家长10%/教师10%/总部20% | 分销关系归因永久有效 |
| LB-P-08 | 投诉闭环 🆕 | P1 | 家长/教师提交投诉→后台处理→反馈 | 48小时SLA，超时升级 |

| LB-T-02 | 需求大厅 | P0 | 教师查看可抢需求列表 | 按科目/年级/地域筛选 |
| LB-T-03 | 我的接单 | P0 | 教师查看已接订单 | 列表+详情+状态跟踪 |
| LB-T-04 | 教师资料管理 | P1 | 教师维护个人资料和教学经历 | 后台审核后展示 |
| LB-T-05 | 教师分销 | P1 | 教师成为分销员，邀请注册获佣金 | 二级分销体系 |
| LB-T-06 | 教师信用体系 🆕 | P1 | S/A/B/C四级信用，影响接单权重 | 信用分=基础60+完成+5-投诉-10-退单-8 |
| LB-T-07 | 教师资质审核 🆕 | P1 | 身份证+学历+资格证审核 | 3次驳回冻结30天 |
| LB-T-08 | 教师提现 🆕 | P1 | 课时费提现，≥100元，T+3到账 | 后台审核通过后打款 |

### 7.1 高优待确认

| 编号 | 问题 | 影响 | 建议 |
|------|------|------|------|

| 风险 | 概率 | 影响 | 缓解措施 |
