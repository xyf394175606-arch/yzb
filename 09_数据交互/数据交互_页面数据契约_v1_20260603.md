## 关键结论

- 页面数据契约按 `领榜教育` 与 `优坐标` 两个独立平台拆分，后台数据域、权限域、分销财务域均独立。
- 领榜核心数据对象为：需求单、抢单记录、接单支付结果、企微监管记录、异常处理记录、客服推送提醒记录、佣金记录、提现申请。
- 优坐标核心数据对象为：老师、产品、老师关联产品、产品订单、会员卡订单、会员权益、约课记录、老师分配记录、替换确认记录、上课安排、佣金记录、提现申请。
- 分销比例、注册奖励、课消提成不写死数值，均以运营配置项返回配置状态与展示文案；原型只展示“按后台配置计算”。
- 分销佣金状态按已确认规则建模：支付成功生成候选佣金；订单完成或课消完成后转为可结算；可结算佣金可进入提现；退款或退单时佣金失效或冲正。
- 提现审核权限按已确认规则建模：财务可审核、驳回、确认打款；管理员可查看和复核；客服无提现审核权限。
- 领榜老师付费购买金额按需求单金额的 `3 倍课时费` 展示和计算。
- 领榜无人抢单不做系统派单，只支持客服人工发送推送消息给老师并记录推送结果。
- 领榜老师抢单资格已确认：老师需提交认证资料；认证成功后可正常接单；未认证老师允许查看未被接单需求单并付款接单，前端必须提醒其进行认证，但不阻断接单。
- 退款人工处理责任已确认：退款责任和财务归属均为系统外事项，当前系统不处理退款责任、退款金额或财务归属。
- 分销归因有效期已确认：拉新注册归因永久有效；销售分销不设置有效期，购买时通过哪个分享链接购买即归属该分享人；无销售分享人时销售提成归属于注册归因人。
- 客服推送老师规则已确认：渠道包含微信服务号、企业微信机器人、短信通知；频率暂按 3 天一次；推送范围为符合需求单要求的老师，例如性别、科目、年级。
- 客服推送老师数量上限尚未确认：当前仅按已确认的渠道、3 天一次频率和匹配老师范围建模，单次/每日推送数量作为后台配置项或后续确认项，不写死数值。
- 第三方排课后续对接方式已确认：通过 API 接口形式对接；当前保留最小排课闭环，并预留 API 同步、失败、重试和冲突状态。
- 优坐标会员卡权益已确认：会员卡只属于优坐标；作用是获得会员购课权益，例如课程对外售价 199，会员价 129。会员分普通会员和卓越会员；普通会员有优惠购课次数限制；卓越会员有效期 1 年，不限制购课数量，但一次给 100 节课时购买机会，用完后可联系客服补充。

| 佣金可结算 | 按完成或课消事件转状态 | 订单完成或课消完成 | 可结算/可提现 | 候选 -> 可结算/可提现 | 退款/退单失效或冲正 | 课消提成仅一级 |
